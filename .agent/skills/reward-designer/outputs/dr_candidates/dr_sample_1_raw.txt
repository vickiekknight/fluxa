```python
# Reasoning for each parameter:

# joint_friction_range:
# Friction in real-world robot joints can vary due to wear, lubrication, and temperature.
# Randomizing friction can help the policy learn to compensate for these variations,
# leading to smoother and more consistent movements. A significant portion of the
# provided range is useful to capture potential real-world friction levels.
joint_friction_range = [0.1, 3.0]

# joint_armature_range:
# Armature represents added inertia that can be due to unmodeled dynamics or mass
# distribution changes in the real world (e.g., attached tools, wear and tear).
# Randomizing armature can make the policy robust to such variations in the robot's
# inertial properties. A moderate range is selected to reflect plausible changes.
joint_armature_range = [0.0, 0.5]

# joint_stiffness_scale_range:
# Actuator stiffness (Kp) can vary in real-world due to factors like motor
# temperature, internal component degradation, and control loop tuning.
# Randomizing stiffness allows the policy to adapt to these variations, ensuring
# it can still reach and maintain the target pose accurately. A range that includes
# deviations from the default is beneficial.
joint_stiffness_scale_range = [0.7, 1.3]

# joint_damping_scale_range:
# Similar to stiffness, actuator damping (Kd) can also vary in real-world.
# Damping is crucial for controlling oscillations and ensuring smooth motion.
# Randomizing damping helps the policy learn to achieve smooth end-effector
# movements even when the damping characteristics of the actuators change.
# A range that covers variations around the default is appropriate.
joint_damping_scale_range = [0.7, 1.3]
```
